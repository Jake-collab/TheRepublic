APP INSTALL POPUP / OPEN-IN-APP BANNER MITIGATION


Many third-party websites display:
- “Download our app”
- “Open in app”
- App Store banners
- Sticky install prompts
- Mobile app overlays


The Republic should minimize these interruptions whenever technically and legally possible inside the WebView experience.


Implement a configurable mitigation system.


Requirements:


- support per-website custom user agents
- support injected CSS
- support injected JavaScript
- support configurable DOM selectors to hide app-install banners
- support optional delayed selector removal after page load
- support optional MutationObserver logic for dynamically inserted popups


Examples of elements to hide:
- “Open in App”
- “Get the App”
- App Store smart banners
- sticky install prompts
- full-page mobile-app overlays
- install bottom sheets


IMPORTANT:
- do NOT break website functionality
- do NOT bypass authentication/security
- do NOT interfere with payment systems
- do NOT manipulate protected/private content


If a website prevents modification because of:
- cross-origin limitations
- CSP restrictions
- anti-tampering systems
- dynamically protected overlays


then fail gracefully without crashing the WebView.


ADMIN PANEL CONTROLS:
Admins should be able to configure per-website:
- app_banner_hide_selectors
- injected_css
- injected_js
- custom_user_agent
- popup mitigation enabled/disabled
- notes about compatibility


The goal is to make websites feel cleaner and more native inside The Republic app while respecting website functionality and security.




PERSISTENT WEBSITE SESSION & TAB MEMORY SYSTEM

The Republic should behave like a persistent multi-tab mobile browser experience.

When users open websites inside tabs:
- preserve their login session whenever technically possible
- preserve cookies/session storage using normal WebView behavior
- preserve the last visited page/URL
- preserve scroll position when possible
- preserve tab state when switching between websites
- avoid unnecessary reloads

TAB MEMORY:
When users switch between website tabs:
- do NOT fully reset the website
- preserve the current page they were on
- preserve the current browsing session
- preserve login state if the website allows persistent sessions

When users close and reopen the app:
- restore their previous website tabs
- restore the last active website tab
- restore the last visited URL for each tab if allowed
- restore expanded/fullscreen mode state if appropriate

SESSION STORAGE:
Use safe/local storage approaches for:
- last visited URLs
- tab ordering
- website preferences
- tab state metadata

Do NOT:
- store passwords
- capture private form fields
- scrape protected content
- bypass website authentication/security

WEBVIEW REQUIREMENTS:
- use persistent WebView cookie/session behavior
- do not unnecessarily destroy inactive tabs
- intelligently unload older tabs only during memory pressure
- preserve tab/browser state whenever possible

USER EXPERIENCE GOAL:
The Republic should feel like:
- a persistent customizable browser
- users should be able to return later and continue where they left off
- websites should feel “still open” when revisiting tabs
- switching tabs should feel smooth and nearly instant
